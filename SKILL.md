# Investment Masters Skill

六位大师平权投票（各 ~16.7%），并行独立分析，输出中文裁决报告。

## 执行流程

### Step 1：数据准备

**行情抓取**（优先用脚本，比搜索更准确）：
```bash
python3 {workspace}/investment-masters/scripts/fetch_price.py GOOGL 700 BTC GOLD
```
> 主数据源：腾讯 qt.gtimg.cn（实时行情，含 PE/市值）。备用：stooq.com（历史 K 线）。常用映射：`700`→港股腾讯，`NVDA`→美股英伟达，`BTC`→IBIT（比特币ETF），`GOLD`→GLD（黄金ETF）。

**基本面数据**：web_search 补充 Forward PE、分析师目标价、近期财报、宏观事件。

**关键信息清单**（装进所有委员 prompt）：
- 价格 + 日涨跌 + 52周区间
- TTM PE / Forward PE（股票）或恐惧贪婪指数（加密）
- RSI / 与200日均线偏差
- 近期重大事件（财报日期、政策、宏观催化剂）
- 若用户有持仓：持仓比例 + 是否已盈亏

### Step 2：识别分析模式

| 用户请求 | 模式 | 重点 |
|---|---|---|
| "分析 XXX" / "要不要买" | **买入判断模式** | 六委员给买入评分 |
| "我持有 X 仓位，怎么办" | **仓位管理模式** | 六委员给操作建议（止损/持有/加仓） |
| "XXX 和 YYY 哪个更好" | **比较模式** | 两标的同时分析，最后比较结论 |

### Step 3：并行召唤六位委员

使用 6 个独立的 sub-agent（Agent 工具），同时发出，确保各自的 prompt 只包含对应大师的 reference 文件内容，不包含其他大师的输出。

每个 sub-agent 的 task 格式：
```
你现在扮演 [大师姓名]。

以下是你的投资哲学和分析框架：
[读取对应 references/ 文件全文]

[买入判断模式] 请用你的视角分析以下标的，严格按你的框架输出评分和判断：
[仓位管理模式] 用户当前 [X%] 仓位持有该标的，请从你的框架出发给出仓位管理建议：
[比较模式] 请用你的视角分别分析以下两个标的，给出各自评分和判断：

标的：[XXX]
当前价格：[YYY]
关键数据：[数据清单]
近期事件：[事件]

输出格式：
评分：X/10
核心判断：2-3 句
关键风险：1 句
操作建议：1 句
```

**关键约束**：六个 sub-agent 必须独立，不能互相看到结论。每个 sub-agent 只接收自己的大师 reference + 数据清单，不接收其他大师的任何输出。

### Step 4：整合裁决

收齐六份结果后，读取 `references/verdict.md` 进行整合：

1. 计算加权综合分（各 ~16.7%）
2. 识别最大分歧（分差最大的两位委员，标注分歧焦点）
3. 检查否决规则（硬否决 / 软否决）
4. 提取可直接执行的操作参数（止损价、仓位、加仓条件）
5. 按 `assets/report-template.md` 格式输出
6. 存档到 `Documents/investment-masters/history/YYYY-MM-DD_标的_说明.md`
